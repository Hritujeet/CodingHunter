How to learn React

React Tutorial in Hindi


In this tutorial, I will be teaching you ReactJS from starting to end. But before deep-diving into the coding part, let me give you a quick introduction of ReactJS and why you should use it.


Introduction :


ReactJS is a JavaScript library originally developed by Facebook that helps in building highly engaging single-page web apps. Many developers call ReactJS a framework instead of a JS library because it is directly comparable with AngularJS. The reason why ReactJS has gained so much popularity in the past years is its simplicity. You need minimal knowledge of JavaScript and HTML. Another thing that helps ReactJS stand out is breaking down the complex UI into simpler components. Today, Facebook’s codebase is containing more than 20,000 React components. Big tech giants such as PayPal, Instagram, Uber, WhatsApp, Yahoo, etc., are using ReactJS to solve their complex UI problems.  


So, we’ve talked enough about the awesomeness of ReactJS. It’s time to get started with the coding part. We will be creating a To-Dos list project with ReactJS. I will be covering the most fundamental aspects & techniques of ReactJS that will help you to create more complex projects in the future.


 


 


Installation :


Node & NPM Installation :


  1. To install ReactJS on your system, you first need to install NodeJS and npm. NodeJS is the platform needed for ReactJS, and npm, which stands for Node Package Manager, will help you to install and manage various JS packages. The good news is that you don’t need to install NodeJS and npm separately. Once you install NodeJS, npm will be automatically installed on your system.
  2. Click here, and you’ll be redirected to the official download page of NodeJS. Download the latest version of NodeJS for your operating system.
  3. Install the NodeJS on your system like any other piece of software.
  4. Once the installation is done, open the Windows PowerShell and type node -v. If the version number is displayed, then NodeJS is successfully installed on your system. Similarly, you can check for npm by typing :

npm -v

Basic Folder Structure :


On opening your React app in VS code, you will see a bunch of files. Here is a quick explanation of some important files you’ll be interacting with :


  1. Public/index.html: This is the main and only HTML file of our React app. This is the page that will be loaded on starting the application.
  2. src/index.js: This is the JavaScript file corresponding to the index.html file, and it is responsible for routing to React components.
  3. src/App.js: App.js is the main component of any React app.
  4. src/App.css: This file is responsible for injecting styling in the React app.

Components In React :


  • Any React app remains divided into several components.
  • Components are nothing but reusable JavaScript functions.
  • Complex UIs can be created easily by breaking them into several components.
  • Every component in React works independently.
  • Even if the components do not depend on each other, they merge inside a parent component to produce the final UI.

Need Of Components :


You might be thinking, why do we need components? Why can't we simply write all the code of a webpage inside a single file? Well! This is what we were doing before React. Before React, developers used to follow the traditional DOM structure to create web pages. But, there are some problems with the traditional approach, like lack of reusability of code or if any mistake is found, it is a challenging task to update because the developer needs to search manually and then update accordingly. But, this is not the case with React, if any mistake is found, we can easily update the code of a particular component.


So, now we're aware of the concept and need of components in React. So, let's get started with creating the first component for our React app.

Default Props :


When we talked about props, I told you that the parent component passes the attributes to the child component. We've also seen that how we passed title from the parent to Header component. But, what will happen if the parent component does not pass any attribute to the child component? In such cases, we will see undefined in the places of the attributes not sent by the parent component. Nobody wants to display undefined on the app because it will show the user that our app is suspicious or not working correctly. To deal with such cases, React comes with the concept of Default Props.


As the name suggests, Default Props will set the default values for the props attributes if the parent component does not send the value. Let's see how it's done :


Header.defaultProps={
  title: "Your title here"
}

Copy


Type the above code at the end of the Header.js file. In the above code, we've set the default value of the title attribute as "Your title here". So, if the parent component does not pass the value of the title attribute, then "Your title here" will be displayed to the user. Open the App.js and do not pass any value for the title attribute as shown in the below code :


import './App.css';
import Header from './MyComponents/Header'

function App() {
  return (
    <>
    <Header/>
    </>
  );
}

export default App;

Copy


Output :



In the above image, it is clearly visible that "Your title here" is rendered when we've not passed any value for the title attribute from the parent component. So, that's how you can easily set the default values for the props attributes and increase the reusability of your code.

Fetching Todo Items From LocalStorage :


Open the App.js and type the below code :


import './App.css';
import Header from "./MyComponents/Header";
import { Todos } from "./MyComponents/Todos";
import { Footer } from "./MyComponents/Footer";
import { AddTodo } from "./MyComponents/AddTodo";
import { About } from "./MyComponents/About";
import React, { useState, useEffect } from 'react';
import {
  BrowserRouter as Router,
  Switch,
  Route
} from "react-router-dom";

function App() {
  let initTodo;
  if (localStorage.getItem("todos") === null) {
    initTodo = [];
  }
  else {
    initTodo = JSON.parse(localStorage.getItem("todos"));
  }


  const onDelete = (todo) => {
    console.log("I am ondelete of todo", todo);
    // Deleting this way in react does not work
    // let index = todos.indexOf(todo);
    // todos.splice(index, 1);

    setTodos(todos.filter((e) => {
      return e !== todo;
    }));
    console.log("deleted", todos)
    localStorage.setItem("todos", JSON.stringify(todos));
  }

  const addTodo = (title, desc) => {
    console.log("I am adding this todo", title, desc)
    let sno;
    if (todos.length === 0) {
      sno = 0;
    }
    else {
      sno = todos[todos.length - 1].sno + 1;
    }
    const myTodo = {
      sno: sno,
      title: title,
      desc: desc,
    }
    setTodos([...todos, myTodo]);
    console.log(myTodo);
  }

  const [todos, setTodos] = useState(initTodo);
  useEffect(() => {
    localStorage.setItem("todos", JSON.stringify(todos));
  }, [todos])

  return ( 
    <> 
    <Router>
      <Header title="My Todos List" searchBar={false} /> 
      <Switch>
          <Route exact path="/" render={()=>{
            return(
            <>
            <AddTodo addTodo={addTodo} />
            <Todos todos={todos} onDelete={onDelete} /> 
            </>)
          }}> 
          </Route>
          <Route exact path="/about">
            <About />
          </Route> 
        </Switch> 
      <Footer />
    </Router>
    </>
  );
}

export default App;

Copy


Now, create a new file named AddTodo.js file. This is the corresponding javascript file for adding components to the todo list. Type the below code :


import React, { useState } from 'react';

export const AddTodo = ({ addTodo }) => {
    const [title, setTitle] = useState("");
    const [desc, setDesc] = useState("");


    const submit = (e) => {
        e.preventDefault();
        if (!title || !desc) {
            alert("Title or Description cannot be blank");
        }
        else {
            addTodo(title, desc);
            setTitle("");
            setDesc("");
        }
    }
    return (
        <div className="container my-3">
            <h3>Add a Todo</h3>
            <form onSubmit={submit}>
                <div className="mb-3">
                    <label htmlFor="title" className="form-label">Todo Title</label>
                    <input type="text" value={title} onChange={(e) => setTitle(e.target.value)} className="form-control" id="title" aria-describedby="emailHelp" />

                </div>
                <div className="mb-3">
                    <label htmlFor="desc" className="form-label">Todo Description</label>
                    <input type="text" value={desc} onChange={(e) => setDesc(e.target.value)} className="form-control" id="desc" />
                </div>
                <button type="submit" className="btn btn-sm btn-success">Add Todo</button>
            </form>
        </div>
    )
}
